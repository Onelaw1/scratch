# 콘텐츠 검토 워크플로우 (Content Review)

리포트, 문서, 데이터 파일 등을 생성했을 때, 사용자가 파일을 열지 않고도 내용을 파악할 수 있게 합니다.

1.  **파일 내용 읽기**
    *   생성된 파일을 `read_file`로 읽습니다.

2.  **핵심 내용 발췌**
    *   전체 내용이 길다면, 가장 중요한 부분(요약, 결론, 혹은 수정된 부분)을 10~20줄 정도 발췌합니다.
    *   특히 **한글**로 된 내용이 있다면 우선적으로 보여줍니다.

3.  **결과 보고 (Walkthrough 업데이트)**
    *   `walkthrough.md`에 발췌한 내용을 인용구로 삽입합니다.
        ```markdown
        > **생성된 리포트 요약:**
        >
        > (발췌한 내용...)
        ```
    *   필요하다면 `carousel` 기능을 활용하여 여러 섹션을 보여줍니다.

> **목표**: 사용자가 "파일 열어서 내용 확인해봐야지"라는 생각이 들지 않게 하는 것입니다.
