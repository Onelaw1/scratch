# 시각적 검증 워크플로우 (Visual Verification)

UI(HTML, CSS, JS)를 수정하거나 새로운 페이지를 만들었을 때, 이 워크플로우를 따라 사용자가 결과물을 눈으로 확인할 수 있게 합니다.

1.  **서버 상태 확인**
    *   백엔드 서버(FastAPI 등)가 실행 중인지 확인합니다.
    *   실행 중이 아니라면 백그라운드로 실행합니다. (`run_command` 사용)

2.  **브라우저 접속 및 캡처**
    *   `browser_subagent`를 사용하여 수정된 페이지 URL로 접속합니다.
    *   페이지가 완전히 로드될 때까지 기다립니다.
    *   **스크린샷**을 촬영합니다. (파일명은 `page_name_verification` 등으로 명확하게)

3.  **결과 보고 (Walkthrough 업데이트)**
    *   `walkthrough.md` 파일을 엽니다.
    *   촬영한 스크린샷을 이미지 태그로 삽입합니다.
        ```markdown
        ![화면 캡처](/path/to/screenshot.png)
        ```
    *   변경 사항이 화면에 잘 반영되었는지 짧은 코멘트를 남깁니다.

> **주의**: 사용자가 "확인해줘"라고 말하지 않아도, UI를 건드렸다면 **자동으로** 이 절차를 수행하는 것이 원칙입니다.
