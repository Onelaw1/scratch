06. 직무평가 (Job Evaluation)

기존 방식의 문제점 (Pain Points)

  • ✖
    평가자 관대화 경향:
    "우리 팀원 챙겨주기" 식으로 전원 상위 점수 부여, 변별력 상실.
  • ✖
    주관적 판단 개입:
    데이터 없이 "힘든 부서"라는 막연한 이미지로 평가.
  • ✖
    단일 평가의 한계:
    부서장 1인의 평가에 의존하여 공정성 시비 발생.

본 시스템의 해결책 (Solution)

  • ✔
    관대화 사전 차단 (Anti-Leniency):
    평가 점수를 표준정규분포(Z-Score)로 자동 변환하여 개인별 편차 제거.
  • ✔
    데이터 기반 검증:
    업무량, 난이도, 책임성 데이터를 화면에 띄워놓고 평가 (Evidence-based).
  • ✔
    멀티 레이터(Multi-Rater) 가중치:
    내부(30%) + 직상위(30%) + 차상위(20%) + 외부위원(20%) 합산.

01

직무조사 (Job Survey)

기존 방식 (Excel/Paper) 본 시스템 (Web-Based)
자유 서술형으로 작성하여 데이터 표준화 불가 AI 표준 과업(Task) 자동 추천 및 선택 방식
작성-취합-검토에 2주 이상 소요 실시간 입력 현황 대시보드 제공, 3일 내 완료
오기입, 누락 발생 시 수기 검증 입력 단계에서 Validation Rule (필수값, 글자수) 자동 체크
02

직무분류 (Classification)

기존 방식 본 시스템
부서 단위로 뭉뚱그려 분류 NCS 국가직무능력표준 기반 세분화
유사 직무 식별 어려움 유사도 분석 알고리즘으로 동일 가치 직무 그룹핑
03

업무량조사 (Workload)

기존 방식 본 시스템
"바쁘다"는 주관적 호소에 의존 Data 산업 평균 소요시간(Standard Time)과 비교 검증
총량적 접근 (연간 2000시간 등) 빈도(Frequency) x 시간(Time) = FTE 자동 산출
검증 불가능 이상치(Outlier) 자동 탐지 (평균 대비 ±30% 이탈 시 경고)
05

정원산정 (Headcount)

Superiority Point

기존의 'Top-down 총액인건비 통제' 방식에서 벗어나, 'Bottom-up 업무량 기반 산정' 결과를 제시함으로써 기재부/주무부처와의 정원 협상에서 압도적인 논리적 우위를 점할 수 있습니다.

12

교육관리 설계 (Training)

기존 방식 본 시스템
직무와 무관한 공통 교육 위주 직무 역량 갭(Gap) 분석 기반 맞춤형 교육 추천
교육 이력 단순 기록 교육 이수 후 역량 향상도(Pre-Post) 측정 및 승진 반영
